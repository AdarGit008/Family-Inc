## Gotchas carried from this session

- **`Finance-Transactions` REQUIRES its header row (row 1).** If it's ever cleared,
  `read_column` goes blind → dedup fails → the tab silently **doubles** on each run.
  (Backlog hardening item: `append_rows` should re-write a missing header.) Don't clear
  the header.
- The budget `SUMIFS` rely on **column order** (Date A / Amount D / Category E) — keep it.
- A Mizrahi run can fail loud on a forced **password change** (`TIMEOUT … #/change-pass`);
  clear it by hand in a browser + update `bank_creds.json` (`FINANCE.md §0`).
